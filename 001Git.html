git init

Initializes a new Git repository in your local folder.

git add .

Stages all files in the current folder for the next commit.

git commit -m "Added 5 more files"

Creates a snapshot of staged changes with a message.

git remote add origin https://github.com/your-username/your-repo-name.git

Links your local repo to a remote GitHub repository.

git remote -v

Shows the remote URLs connected to your local repo.

git remote set-url origin https://github.com/newusername/new-repo.git

Change Remote Repository

git push -u origin main

Pushes your main branch to GitHub and sets it as default upstream.

Pushes and sets a link (so next time you can just type git push or git pull).

origin - Default name for your GitHub remote - Shortcut to GitHub URL

Use -u the first time you push a branch.

git push origin main

Pushes code manually to remote. If branch already linked. It does not remember this connection.

you will always need to type origin main each time.

git push -u origin master

Pushes the master branch to GitHub (only if it still exists locally).

git branch

Lists all local branches and highlights the current one.

main - Default branch name (new standard) - Inclusive, consistent, modern
master - Old default branch name - Still works, but not preferred now

git branch -m master main

Renames the current branch from master to main.

git status

Shows the current state of your working directory and staging area.
